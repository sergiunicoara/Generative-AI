API Documentation - User Authentication Service

## Overview
The User Authentication Service provides secure authentication and authorization for all applications.

## Authentication Methods

### OAuth 2.0
OAuth 2.0 is the recommended authentication method for third-party applications.

Endpoint: POST /api/v1/oauth/token
Content-Type: application/json

Request Body:
{
  "grant_type": "client_credentials",
  "client_id": "your_client_id",
  "client_secret": "your_client_secret"
}

Response:
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600
}

### API Key Authentication
For server-to-server communication, use API key authentication.

Header: X-API-Key: your_api_key

Example:
curl -H "X-API-Key: sk_live_abc123" https://api.example.com/v1/users

### JWT Authentication
JSON Web Tokens are used for user session management.

Endpoint: POST /api/v1/auth/login
Content-Type: application/json

Request:
{
  "email": "user@example.com",
  "password": "secure_password"
}

Response:
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": "usr_123",
    "email": "user@example.com",
    "name": "John Doe"
  }
}

## Rate Limiting

API Rate Limits:
- Free Tier: 100 requests/hour
- Basic Tier: 1,000 requests/hour
- Pro Tier: 10,000 requests/hour
- Enterprise: Unlimited

Rate limit headers:
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 750
X-RateLimit-Reset: 1640995200

## Error Codes

400 Bad Request - Invalid request parameters
401 Unauthorized - Missing or invalid authentication
403 Forbidden - Insufficient permissions
404 Not Found - Resource not found
429 Too Many Requests - Rate limit exceeded
500 Internal Server Error - Server error

## Best Practices

1. Always use HTTPS for API calls
2. Store API keys securely (use environment variables)
3. Implement proper error handling
4. Use refresh tokens for long-lived sessions
5. Rotate API keys regularly
6. Monitor API usage and rate limits

## Code Examples

### Python
import requests

url = "https://api.example.com/v1/users"
headers = {
    "X-API-Key": "sk_live_abc123",
    "Content-Type": "application/json"
}

response = requests.get(url, headers=headers)
if response.status_code == 200:
    users = response.json()
    print(users)
else:
    print(f"Error: {response.status_code}")

### JavaScript
const axios = require('axios');

const config = {
  headers: {
    'X-API-Key': 'sk_live_abc123',
    'Content-Type': 'application/json'
  }
};

axios.get('https://api.example.com/v1/users', config)
  .then(response => {
    console.log(response.data);
  })
  .catch(error => {
    console.error('Error:', error);
  });

### cURL
curl -X GET \
  https://api.example.com/v1/users \
  -H 'X-API-Key: sk_live_abc123' \
  -H 'Content-Type: application/json'
